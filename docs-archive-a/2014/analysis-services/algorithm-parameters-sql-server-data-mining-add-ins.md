# <a name="algorithm-parameters-sql-server-data-mining-add-ins"></a>Параметры алгоритма (надстройки интеллектуального анализа данных SQL Server)
  Если интеллектуальный анализ данных выполняется с использованием средств анализа таблиц для Excel, нет необходимости настраивать алгоритм интеллектуального анализа данных или параметры. Каждое средство анализирует предоставленные данные и автоматически выбирает оптимальные параметры. Однако если потребуется изменить модель или создать модель интеллектуального анализа данных с нуля, то клиент интеллектуального анализа данных для Excel предоставляет несколько параметров для настройки.  
  
-   Создайте модель интеллектуального анализа данных вручную, нажав кнопку **Дополнительно** и выбрав **Добавить модель в структуру**.  
  
-   Используйте любой из мастеров моделирования в клиенте интеллектуального анализа данных и щелкните **Параметры** , чтобы управлять поведением [!INCLUDE[msCoName](../includes/msconame-md.md)] алгоритмов интеллектуального анализа данных.  
  
-   Нажмите кнопку **запрос** , чтобы открыть мастер моделей запросов, а затем нажмите кнопку **Дополнительно** , чтобы открыть **Расширенный редактор запросов интеллектуального анализа данных**. Этот редактор предоставляет возможность строить модели с использованием шаблонов расширений интеллектуального анализа данных.  
  
 Можно также изменить поведение уже созданной модели интеллектуального анализа данных или отфильтровать результаты, установив параметры в средстве просмотра моделей интеллектуального анализа данных.  
  
## <a name="list-of-algorithm-parameters"></a>Список параметров алгоритма  
 Все [!INCLUDE[msCoName](../includes/msconame-md.md)] алгоритмы можно настраивать, задавая параметры. Наиболее подходящие значения параметров зависят от состава применяемых данных, поэтому полное описание того, к чему приводит изменение параметров, выходит за рамки настоящего раздела.  
  
 Ниже приведена таблица, в которой перечисляются параметры, описаны их функциональные возможности и предоставлены ссылки на более подробные технические сведения.  
  
|Имя параметра|Используется в|Описание|  
|--------------------|-------------|-----------------|  
|AUTO_DETECT_PERIODICITY|Алгоритм временных рядов (Майкрософт)|Указывает числовое значение от 0 до 1, используемое для обнаружения периодичности. Установка этого значения, близкого к 1, повышает вероятность обнаружения многих шаблонов, близких к периодическим, и автоматическим формированием подсказок периодичности. Использование большого количества подсказок периодичности, скорее всего, значительно увеличит время обучения модели, а также ее точность. Если значение близко к 0, то периодичность обнаруживается только для строго периодических данных.<br /><br /> Значение по умолчанию равно 0,6.|  
|CLUSTER_COUNT|Алгоритм кластеризации (Майкрософт)<br /><br /> Алгоритм кластеризации последовательностей (Майкрософт)|Указывает примерное количество кластеров, строящихся данным алгоритмом. Если это примерное количество кластеров не может быть построено из данных, то алгоритм строит столько кластеров, сколько возможно. Значение 0 параметра CLUSTER_COUNT приводит к тому, что алгоритм начинает использовать эвристический подход для наиболее эффективного определения числа строящихся кластеров.<br /><br /> Значение по умолчанию равно 10.|  
|CLUSTER_SEED|Алгоритм кластеризации (Майкрософт)|Указывает начальное значение, используемое для случайного формирования кластеров в начальной стадии построения модели.<br /><br /> Значение по умолчанию равно 0.|  
|CLUSTERING_METHOD|Алгоритм кластеризации (Майкрософт)|Указывает метод кластеризации, используемый алгоритмом. Доступны следующие методы кластеризации: масштабируемая максимизация ожидания (1), немасштабируемая МАКСИМИЗАЦия ожидания (2), масштабируемые K-средние (3) и немасштабируемые K-средние (4).<br /><br /> Значение по умолчанию — 1.|  
|COMPLEXITY_PENALTY|Алгоритм дерева принятия решений (Майкрософт)<br /><br /> Алгоритм временных рядов (Майкрософт)|Управляет ростом дерева решений. Низкое значение увеличивает количество разбиений, а высокое количество — уменьшает. Значение по умолчанию основано на количестве атрибутов для конкретной модели, как описано в следующем списке.<br /><br /> Для атрибутов с 1 до 9 значением по умолчанию является 0,5.<br /><br /> Для атрибутов с 10 до 99 значением по умолчанию является 0,9.<br /><br /> Для 100 или более атрибутов значением по умолчанию является 0,99.<br /><br /> Примечание. модели временных рядов этот параметр применяется только к моделям, построенным с помощью алгоритма ARTxp или смешанных моделей.|  
|FORCED_REGRESSOR|Алгоритм дерева принятия решений (Майкрософт)<br /><br /> Алгоритм линейной регрессии (Майкрософт)|Приводит алгоритм к использованию указанных столбцов в качестве регрессоров, не обращая внимания на важность столбцов, вычисленную алгоритмом.<br /><br /> Примечание. Этот параметр используется только для деревьев принятия решений, которые прогнозирует непрерывный атрибут. По определению, модель линейной регрессии представляет собой частный случай деревьев решений и предназначена для прогнозирования непрерывных атрибутов. Но узел, представляющий формулу линейной регрессии, может содержаться в любой модели дерева решений.|  
|FORECAST_METHOD|Алгоритм временных рядов (Майкрософт)|Указывает, должны ли выполняться прогнозы с использованием алгоритма ARTxp, алгоритма ARIMA или их сочетания.<br /><br /> Значением по умолчанию является MIXED.|  
|HIDDEN_NODE_RATIO|Microsoft Neural Network Algorithm|Указывает соотношение скрытых нейронов к входным и выходным нейронам. Следующая формула определяет начальное количество нейронов в скрытом слое:<br /><br /> HIDDEN_NODE_RATIO * SQRT (количество входных нейронов \* количество выходных нейронов)<br /><br /> Значение по умолчанию — 4,0.|  
|HISTORIC_MODEL_COUNT|Алгоритм временных рядов (Майкрософт)|Указывает число моделей с предысторией, которые будут построены.<br /><br /> Значение по умолчанию — 1.|  
|HISTORICAL_MODEL_GAP|Алгоритм временных рядов (Майкрософт)|Указывает временной промежуток между двумя последовательными моделями с предысторией. Например, если установить это значение равным g, модели с предысторией будут строиться для данных, усеченных временными срезами с интервалами g, 2*g, 3\*g и так далее.<br /><br /> Значение по умолчанию равно 10.|  
|HOLDOUT_PERCENTAGE|Алгоритм логистической регрессии (Майкрософт)<br /><br /> Microsoft Neural Network Algorithm|Указывает процент вариантов в составе обучающих данных, используемых для вычисления ошибки контрольных данных, которая применяется как один из критериев остановки во время обучения модели интеллектуального анализа данных.<br /><br /> Значение по умолчанию — 30.<br /><br /> Примечание. Этот параметр отличается от значения контрольных данных в процентах, применяемого к структуре интеллектуального анализа данных.|  
|HOLDOUT_SEED|Алгоритм логистической регрессии (Майкрософт)<br /><br /> Microsoft Neural Network Algorithm|Указывает значение, используемое генератором псевдослучайных чисел в качестве начального, когда алгоритм случайным образом задает контрольные данные. При установке данного параметра равным 0 алгоритм формирует начальное значение на основе имени модели интеллектуального анализа данных, что гарантирует неизменность содержимого модели при повторной обработке.<br /><br /> Значение по умолчанию — 0.<br /><br /> Примечание. Этот параметр отличается от начального контрольного значения, применяемого к структуре интеллектуального анализа данных.|  
|INSTABILITY_SENSITIVITY|Алгоритм временных рядов (Майкрософт)|Управляет тем, в какой точке дисперсия прогноза превышает определенное пороговое значение, а алгоритм ARTxp подавляет прогнозы. Значение по умолчанию — 1.<br /><br /> Примечание. Этот параметр применяется только к смешанным моделям или моделям, использующим алгоритм ARTxp.|  
|MAXIMUM_INPUT_ATTRIBUTES|Алгоритм кластеризации (Майкрософт)<br /><br /> Алгоритм дерева принятия решений (Майкрософт)<br /><br /> Алгоритм линейной регрессии (Майкрософт)<br /><br /> Упрощенный алгоритм Байеса (Майкрософт)<br /><br /> Microsoft Neural Network Algorithm<br /><br /> Алгоритм логистической регрессии (Майкрософт)|Определяет количество входных атрибутов, которые алгоритм может обработать перед вызовом выбора компонентов. Установите значение 0, чтобы отключить выбор компонентов.<br /><br /> Значение по умолчанию — 255.|  
|MAXIMUM_ITEMSET_COUNT|Алгоритм взаимосвязей (Майкрософт)|Указывает максимальное количество создаваемых наборов элементов. Если значение не указано, то алгоритм формирует все возможные наборы элементов.<br /><br /> Значение по умолчанию — 200 000.|  
|MAXIMUM_ITEMSET_SIZE|Алгоритм взаимосвязей (Майкрософт)|Указывает максимальное количество элементов, допустимых в наборе элементов. Задание этого значения равным 0 указывает, что размер набора элементов не ограничен.<br /><br /> По умолчанию используется значение 3.|  
|MAXIMUM_OUTPUT_ATTRIBUTES|Алгоритм дерева принятия решений (Майкрософт)<br /><br /> Алгоритм линейной регрессии (Майкрософт)<br /><br /> Алгоритм логистической регрессии (Майкрософт)<br /><br /> Упрощенный алгоритм Байеса (Майкрософт)<br /><br /> Microsoft Neural Network Algorithm|Определяет количество выходных атрибутов, которые алгоритм может обработать перед вызовом выбора компонентов. Установите значение 0, чтобы отключить выбор компонентов.<br /><br /> Значение по умолчанию — 255.|  
|MAXIMUM_SEQUENCE_STATES|Алгоритм кластеризации последовательностей (Майкрософт)|Указывает максимальное количество состояний, которые последовательность может иметь. Установка данного значения равным числу, большему 100, может привести к тому, что алгоритм создаст модель, не предоставляющую достоверных данных.<br /><br /> Значение по умолчанию — 64.|  
|MAXIMUM_SERIES_VALUE|Алгоритм временных рядов (Майкрософт)|Указывает максимальное значение, используемое для прогнозов. Этот параметр используется наряду с параметром MINIMUM_SERIES_VALUE для ограничения прогнозов с учетом некоторого ожидаемого диапазона. Например, можно указать, что прогнозируемый объем продаж в течение любых суток никогда не должен превышать величину запасов товаров.|  
|MAXIMUM_STATES|Алгоритм кластеризации (Майкрософт)<br /><br /> Microsoft Neural Network Algorithm<br /><br /> Алгоритм кластеризации последовательностей (Майкрософт)|Указывает максимальное количество состояний атрибутов, поддерживаемое алгоритмом. Если число состояний атрибута превышает максимальное число состояний, то алгоритм использует наиболее популярные состояния атрибута и игнорирует оставшиеся состояния.<br /><br /> Значение по умолчанию — 100.|  
|MAXIMUM_SUPPORT|Алгоритм взаимосвязей (Майкрософт)|Указывает максимальное количество вариантов, в которых может поддерживаться набор элементов. Если это значение меньше 1, то оно представляет процент от общего количества вариантов. Значение больше 1 представляет абсолютное количество вариантов, в которых может содержаться набор элементов.<br /><br /> Значение по умолчанию — 1.|  
|MINIMUM_IMPORTANCE|Алгоритм взаимосвязей (Майкрософт)|Указывает пороговое значение важности для правил взаимосвязей. Правила, имеющие важность меньше этого значения, отфильтровываются.|  
|MINIMUM_ITEMSET_SIZE|Алгоритм взаимосвязей (Майкрософт)|Указывает минимальное количество элементов, допустимых в наборе элементов.<br /><br /> Значение по умолчанию — 1.|  
|MINIMUM_DEPENDENCY_PROBABILITY|Упрощенный алгоритм Байеса (Майкрософт)|Задает минимальную вероятность зависимости между входным и выходным атрибутами. Это значение используется для ограничения размера содержимого, формируемого алгоритмом. Для этого свойства можно задать значение от 0 до 1. Большие значения уменьшают количество атрибутов в содержимом модели.<br /><br /> Значение по умолчанию равно 0,5.|  
|MINIMUM_PROBABILITY|Алгоритм взаимосвязей (Майкрософт)|Указывает минимальную вероятность того, что правило имеет значение true. Например, задание этого значения равным 0,5 указывает, что правила с вероятностью меньше 50% не формируются.<br /><br /> Значение по умолчанию — 0,4.|  
|MINIMUM_SERIES_VALUE|Алгоритм временных рядов (Майкрософт)|Указывает нижнее ограничение для всех прогнозов временных рядов. Прогнозируемые значения никогда не будут ниже этого ограничения.|  
|MINIMUM_SUPPORT|Алгоритм взаимосвязей (Майкрософт)|Указывает минимальное количество вариантов, которые должны содержать набор элементов, прежде чем алгоритм создаст правило. Значение меньше 1 указывает минимальное количество вариантов как процент от общего количества вариантов. Целое значение больше 1 задает минимальное количество вариантов как абсолютное количество вариантов, которые должны содержать набор элементов. Алгоритм может увеличить значение этого параметра, если объем памяти ограничен.<br /><br /> Значение по умолчанию — 0,03.|  
|MINIMUM_SUPPORT|Алгоритм кластеризации (Майкрософт)|Указывает минимальное число вариантов в каждом кластере.<br /><br /> Значение по умолчанию — 1.|  
|MINIMUM_SUPPORT|Алгоритм дерева принятия решений (Майкрософт)|Определяет минимальное количество конечных вариантов, необходимых для формирования разбиения в дереве решений.<br /><br /> Значение по умолчанию равно 10.|  
|MINIMUM_SUPPORT|Алгоритм кластеризации последовательностей (Майкрософт)|Указывает минимальное число вариантов в каждом кластере.<br /><br /> Значение по умолчанию равно 10.|  
|MINIMUM_SUPPORT|Алгоритм временных рядов (Майкрософт)|Указывает минимальное количество временных срезов, необходимых для разбиения каждого дерева временных рядов.<br /><br /> Значение по умолчанию равно 10.|  
|MISSING_VALUE_SUBSTITUTION|Алгоритм временных рядов (Майкрософт)|Указывает метод, используемый для заполнения пропусков в данных с предысторией. По умолчанию не допускаются данные с неравномерными пропусками и неровными границами. Для заполнения нерегулярных пропусков или краев можно использовать следующие методы: используйте предыдущее значение, используйте среднее значение или используйте определенную числовую константу.|  
|MODELLING_CARDINALITY|Алгоритм кластеризации (Майкрософт)|Указывает число образцов моделей, создаваемое в процессе кластеризации.<br /><br /> Значение по умолчанию равно 10.|  
|PERIODICITY_HINT|Алгоритм временных рядов (Майкрософт)|Предоставляет подсказку для алгоритма, касающуюся периодичности данных. Например, если продажи варьируются в зависимости от года, а в ряду в качестве единицы измерения используются месяцы, то периодичность равна 12. Этот параметр имеет формат {n [, n]}, где n — любое положительное число. Число n внутри квадратных скобок является необязательным и может повторяться произвольное количество раз.<br /><br /> Значение по умолчанию — {1}.|  
|PREDICTION_SMOOTHING|Алгоритм временных рядов (Майкрософт)|Управляет соотношением смешивания временных рядов по алгоритмам ARTXP и ARIMA. Указанное значение допустимо, только если параметр FORECAST_METHOD установлен в значение MIXED. Значения должны находиться в диапазоне от 0 до 1. Если значение равно 0, то модель использует только алгоритм ARTXP. Если значение равно 1, то модель использует только алгоритм ARIMA. Чем ближе значение к нулю, тем больший вес имеет алгоритм ARTXP. Чем ближе значение к 1, тем больший вес имеет алгоритм ARIMA.|  
|SAMPLE_SIZE|Алгоритм кластеризации (Майкрософт)|Указывает количество объектов, которые этот алгоритм использует при каждом проходе, если для параметра CLUSTERING_METHOD задан один из методов масштабируемой кластеризации. Значение 0 параметра SAMPLE_SIZE приводит к тому, что весь набор данных кластеризуется за один проход. Это может привести к проблемам, связанным с памятью и производительностью.<br /><br /> Значение по умолчанию — 50 000.|  
|SAMPLE_SIZE|Алгоритм логистической регрессии (Майкрософт)<br /><br /> Microsoft Neural Network Algorithm|Указывает количество вариантов, которые будут использоваться для обучения модели. Поставщик алгоритма использует либо это число, либо процент от суммарного количества вариантов, не включенных в процент контрольных данных, в соответствии с параметром HOLDOUT_PERCENTAGE, выбирая наименьшее из этих двух значений.<br /><br /> Другими словами, если значение параметра HOLDOUT_PERCENTAGE установлено равным 30, то алгоритм будет использовать либо значение этого параметра, либо значение, равное 70 процентам от общего количества вариантов — в зависимости от того, которое из двух указанных значений меньше.<br /><br /> Значение по умолчанию — 10000.|  
|SCORE_METHOD|Алгоритм дерева принятия решений (Майкрософт)|Определяет метод, используемый для вычисления коэффициента разбиения. Доступны следующие параметры: (1) энтропия, (2) Байеса с априорной оценкой K2 ранее или (3) Байеса Дирихле метода (BDE).<br /><br /> По умолчанию используется значение 3.|  
|SPLIT_METHOD|Алгоритм дерева принятия решений (Майкрософт)|Определяет метод, используемый для разбиения узла. Доступны следующие параметры: двоичный (1), полный (2) или оба (3).<br /><br /> По умолчанию используется значение 3.|  
|STOPPING_TOLERANCE|Технический справочник по алгоритму кластеризации (Майкрософт)|Указывает значение, используемое для определения момента достижения конвергенции и завершения построения модели. Конвергенция достигается, когда общее изменение вероятностей кластеров становится меньше, чем отношение параметра STOPPING_TOLERANCE к размеру модели.<br /><br /> Значение по умолчанию равно 10.|
